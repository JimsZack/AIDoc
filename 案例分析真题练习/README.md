# 案例分析真题练习 — 使用说明

> 本目录收录软考系统架构设计师**案例分析（科目二）**的综合案例练习题。
> 每道案例题均提供**完整题目 + 参考答案**，方便对照模板自我评估。

---

## 目录结构

```
案例分析真题练习/
├── README.md                           ← 本文件
├── 01-架构风格综合案例.md               ← 架构风格选型与分析案例
├── 02-质量属性与架构策略案例.md          ← 质量属性识别与架构策略案例
├── 03-微服务架构设计案例.md             ← 微服务拆分与设计案例
└── 04-设计模式综合案例.md               ← 设计模式应用案例
```

## 使用方法

1. **先做题**：遮住参考答案，自己按模板结构组织答案
2. **对照评估**：与参考答案对比，找出遗漏的得分点
3. **套模板练习**：结合[案例分析答题模板](../案例分析答题模板/README.md)，反复练习
4. **总结规律**：总结每种题型的必答要点，形成自己的答题思路

## 评分要点说明

软考案例题采用**踩点给分**机制：
- 每个得分点 1–2 分
- 关键词说对了即得分
- 表达方式不限，专业术语优先
- 多答不扣分

> ⚠️ 参考答案仅供学习参考，不代表唯一正确答案。实际考场以评卷标准为准。
