# 架构风格真题精选与解析

> 架构风格是综合知识中**出题最密集**的知识点之一，几乎每年都有 2–4 题。
> 核心考法：给出场景描述，判断属于哪种架构风格。

---

## 题目 1（高频经典）

某系统需要对输入的原始数据依次进行词法分析、语法分析、语义分析和代码生成，每个阶段的输出作为下一个阶段的输入。该系统最适合采用的架构风格是（ ）。

A. 仓库风格  
B. 管道-过滤器风格  
C. 分层架构风格  
D. 事件驱动风格

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"依次""每个阶段的输出作为下一阶段的输入"** → 数据流顺序经过多个处理节点
- 这是管道-过滤器的典型场景（编译器）
- **排除法**：
  - A 仓库风格强调共享数据存储（如黑板系统），不匹配
  - C 分层架构强调职责分层和单向调用，不强调数据流
  - D 事件驱动强调异步触发，不符合"顺序"特征

### 🎯 秒杀口诀

> 看到"编译器""数据顺序流经多个处理节点""阶段性转换" → **管道-过滤器**

## 题目 2

某智能医疗辅助诊断系统需要集成多个专家子系统的知识，各子系统独立分析并将结果写入共享区域，由控制模块综合判断。该系统最适合的架构风格是（ ）。

A. 分层架构  
B. 客户端-服务器架构  
C. 黑板架构  
D. 管道-过滤器架构

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"多个专家子系统""共享区域""综合判断"**
- 黑板架构的三要素：**知识源（多个专家）+ 黑板（共享区域）+ 控制器（综合判断）**
- **排除法**：
  - A 分层不涉及"多知识源协作"
  - B C/S 是请求-响应模式
  - D 管道-过滤器是顺序数据流，不是协作式

### 🎯 秒杀口诀

> 看到"AI 推理""专家系统""多知识源""共享协作" → **黑板架构**

---

## 题目 3

某企业管理信息系统将功能组织为表现层、业务逻辑层和数据访问层，上层只能调用相邻下层的服务。该系统采用的架构风格是（ ）。

A. 管道-过滤器  
B. 事件驱动  
C. 分层架构  
D. 微服务架构

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"表现层""业务逻辑层""数据访问层""上层只能调用相邻下层"**
- 这些都是分层架构的标准表述
- **排除法**：
  - A 管道-过滤器强调数据流
  - B 事件驱动强调异步通知
  - D 微服务强调独立部署、API 网关

### 🎯 秒杀口诀

> 看到"表现层/业务层/数据层""单向调用""层间依赖" → **分层架构**

## 题目 4

某互联网电商平台需要将系统拆分为用户服务、订单服务、支付服务等独立模块，每个模块可以独立部署和扩展，模块之间通过 RESTful API 通信。该系统适合采用的架构风格是（ ）。

A. 分层架构  
B. 微服务架构  
C. 管道-过滤器  
D. 黑板架构

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"独立模块""独立部署和扩展""RESTful API 通信"**
- 微服务的核心特征：服务自治 + 独立部署 + 轻量通信
- **排除法**：
  - A 分层架构关注逻辑分层，而非服务拆分
  - C 管道-过滤器关注数据流处理
  - D 黑板架构关注知识协作

### 🎯 秒杀口诀

> 看到"服务拆分""独立部署""API 网关""容错隔离" → **微服务架构**

## 题目 5

某实时交易监控系统需要在用户下单后自动触发风险检测、库存扣减和通知发送，各处理模块之间不直接调用，而是通过消息中间件解耦。该系统最适合的架构风格是（ ）。

A. 分层架构  
B. 微服务架构  
C. 事件驱动架构  
D. 客户端-服务器架构

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"自动触发""不直接调用""消息中间件解耦"**
- 事件驱动架构核心：基于事件发布/订阅，处理模块间异步解耦
- **易错点**：题目中也提到了多个"服务"，但关键判断是它们之间**不直接调用**而是通过事件/消息驱动

### 🎯 秒杀口诀

> 看到"触发""发布/订阅""事件总线""异步通知""消息解耦" → **事件驱动架构**

## 题目 6

在某系统中，用户可以通过自定义脚本语言来配置业务规则，系统内部包含一个解释器来逐条执行这些规则脚本。该系统架构属于（ ）。

A. 黑板架构  
B. 虚拟机架构（解释器风格）  
C. 管道-过滤器  
D. 分层架构

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"自定义脚本语言""解释器""逐条执行规则"**
- 虚拟机架构（解释器风格）核心特征：存在一个解释引擎，将自定义语言/规则翻译执行
- **排除法**：
  - A 黑板架构强调多知识源协作
  - C 管道-过滤器强调数据顺序流转
  - D 分层架构强调层次间单向依赖

### 🎯 秒杀口诀

> 看到"解释器""规则引擎""自定义语言""脚本执行" → **虚拟机风格**

---

## 考点总结：架构风格快速对照表

| 架构风格 | 看到什么关键词就选它 |
|---|---|
| 分层架构 | 表现层/业务层/数据层、单向调用、Web 三层 |
| 管道-过滤器 | 编译器、数据顺序流、阶段性转换 |
| 黑板架构 | AI 推理、专家系统、多知识源、共享数据区 |
| 事件驱动 | 触发、发布/订阅、异步、消息中间件 |
| 微服务 | 独立部署、服务拆分、API 网关、熔断 |
| 虚拟机/解释器 | 脚本语言、规则引擎、解释执行 |
| C/S | 客户端请求、服务端响应、集中管理 |
