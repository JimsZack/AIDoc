# 设计模式真题精选与解析

> 设计模式题的常见考法：
> 1. 给出场景描述，判断适合用哪个模式
> 2. 给出模式名称，问属于创建型/结构型/行为型哪一类
> 3. 给出代码片段，判断是什么模式

---

## 题目 1（分类判断 - 高频）

以下设计模式中，属于结构型模式的是（ ）。

A. 观察者模式  
B. 工厂方法模式  
C. 适配器模式  
D. 策略模式

### ✅ 答案：C

### 📖 解析

- **创建型**（造对象）：单例、工厂方法、抽象工厂、建造者、原型
- **结构型**（组装类）：适配器、桥接、组合、装饰、外观、享元、代理
- **行为型**（管逻辑）：观察者、策略、状态、命令、责任链、模板方法、迭代器、中介者、备忘录、访问者、解释器
- A 观察者 → 行为型
- B 工厂方法 → 创建型
- D 策略 → 行为型

### 🎯 速记口诀

> **创建**：单工建原抽（单例、工厂方法、建造者、原型、抽象工厂）  
> **结构**：适桥组装外享代（适配器、桥接、组合、装饰、外观、享元、代理）  
> **行为**：观策状命责模迭中备访解（观察者、策略、状态、命令、责任链、模板方法、迭代器、中介者、备忘录、访问者、解释器）

## 题目 2（场景判断）

某系统需要记录所有对数据库的操作，以便在出错时能够撤销最近的操作并恢复到之前的状态。最适合使用的设计模式是（ ）。

A. 观察者模式  
B. 命令模式  
C. 策略模式  
D. 单例模式

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"记录操作""撤销""恢复"**
- 命令模式的核心能力：将请求封装为对象，支持撤销（undo）、排队、日志记录
- **排除法**：
  - A 观察者是通知，不管撤销
  - C 策略是算法切换
  - D 单例是保证唯一实例

### 🎯 秒杀口诀

> 看到"撤销""Undo""操作记录""命令队列" → **命令模式**

## 题目 3（场景判断）

某电商系统支持微信支付、支付宝支付、银行卡支付等多种方式，用户可以在下单时选择不同的支付方式。为了方便后续新增支付方式，应该采用的设计模式是（ ）。

A. 单例模式  
B. 适配器模式  
C. 观察者模式  
D. 策略模式

### ✅ 答案：D

### 📖 解析

- 题干关键词：**"多种方式""动态选择""新增方式"**
- 策略模式核心：定义一族算法，封装每个算法，使它们可以互相替换
- **易错点**：适配器用于"接口不兼容转换"，不是"多种等价策略切换"

### 🎯 秒杀口诀

> 看到"多种方式可选""动态切换算法""支付/排序/计费策略" → **策略模式**

## 题目 4（场景判断）

某系统中，当库存数量发生变化时，需要自动通知仓储管理模块、财务模块和报表模块进行相应更新。最适合使用的设计模式是（ ）。

A. 单例模式  
B. 代理模式  
C. 观察者模式  
D. 工厂模式

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"变化时自动通知""多个模块更新"**
- 观察者模式核心：一个对象（Subject）状态变化时，自动通知所有依赖它的对象（Observer）
- 这是典型的一对多通知场景

### 🎯 秒杀口诀

> 看到"状态变化""自动通知""一对多""监听/订阅" → **观察者模式**

---

## 题目 5（场景判断）

某系统需要访问一个远程服务，为了减少网络调用次数，在本地维护了一个代理对象，该代理对象缓存了远程服务的部分数据，只在必要时才发起远程调用。该设计使用了（ ）模式。

A. 装饰模式  
B. 适配器模式  
C. 代理模式  
D. 外观模式

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"代理对象""缓存""控制访问""远程调用"**
- 代理模式核心：为其他对象提供一种代理以控制对这个对象的访问
- 代理的类型：远程代理、虚拟代理（延迟加载）、保护代理（权限控制）

### 🎯 辨析

> - **代理**：控制对原对象的访问，接口和原对象一致
> - **适配器**：转换接口，让不兼容的接口能一起工作
> - **装饰**：动态增强功能，不改变接口
> - **外观**：为复杂子系统提供简化入口

## 题目 6（场景判断）

某旧系统的数据接口返回 XML 格式，而新系统只能处理 JSON 格式。为了让新系统能使用旧系统的数据，开发团队编写了一个转换层。该转换层使用了（ ）模式。

A. 代理模式  
B. 适配器模式  
C. 桥接模式  
D. 策略模式

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"接口不兼容""转换层""让新旧系统能协作"**
- 适配器模式核心：将一个类的接口转换成客户端期望的另一个接口
- 这里"XML 转 JSON"就是接口转换

### 🎯 秒杀口诀

> 看到"旧系统对接""接口不兼容""转换/包装" → **适配器模式**

## 题目 7（场景判断）

某系统中日志管理器需要保证全局只有一个实例，所有模块共享该实例。该设计使用了（ ）模式。

A. 工厂模式  
B. 单例模式  
C. 原型模式  
D. 建造者模式

### ✅ 答案：B

### 📖 解析

- 题干关键词：**"全局只有一个实例""所有模块共享"**
- 单例模式核心：确保一个类只有一个实例，并提供全局访问点

## 考点总结：常考模式场景对照

| 设计模式 | 典型场景关键词 |
|---|---|
| 单例 | 全局唯一、共享实例、日志管理器 |
| 工厂方法 | 创建对象时不指定具体类、由子类决定创建 |
| 抽象工厂 | 创建一族相关产品 |
| 观察者 | 状态变化通知、一对多、监听/订阅 |
| 策略 | 多种算法/方式可选、动态切换 |
| 命令 | 撤销、操作记录、命令队列 |
| 适配器 | 接口不兼容转换、旧系统对接 |
| 代理 | 控制访问、远程代理、缓存、权限检查 |
| 装饰 | 动态增加功能、不修改原有结构 |
| 外观 | 简化复杂子系统调用、统一入口 |
| 模板方法 | 算法骨架固定、细节由子类实现 |
| 责任链 | 请求逐级传递、审批流程 |
