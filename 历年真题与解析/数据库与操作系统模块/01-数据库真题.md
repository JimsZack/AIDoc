## 一、事务 ACID

### 题目 1（经典高频）

在数据库事务中，"一个事务中的所有操作要么全部成功，要么全部回滚"，这体现了事务的（ ）特性。

A. 一致性  
B. 隔离性  
C. 原子性  
D. 持久性

### ✅ 答案：C

### 📖 解析

- ACID 四个特性口诀：**原（子性）一（致性）隔（离性）持（久性）**
- **原子性**：事务是一个不可分割的工作单位，要么全做，要么全不做
- **易错点**：不要把"全部成功"误理解为"一致性"。一致性强调的是事务前后数据状态的正确性

| 特性 | 关键词 |
|------|--------|
| 原子性 | 要么全做要么全不做、回滚 |
| 一致性 | 数据正确、完整性约束不被破坏 |
| 隔离性 | 并发事务互不干扰 |
| 持久性 | 提交后永久生效、不会丢失 |

---

### 题目 2

数据库系统使用**重做日志（Redo Log）**主要是为了保证事务的（ ）。

A. 原子性  
B. 一致性  
C. 隔离性  
D. 持久性

### ✅ 答案：D

### 📖 解析

- 重做日志（Redo Log）的作用：在系统崩溃后可以通过重放日志来恢复已提交事务的修改，保证持久性
- **撤销日志（Undo Log）** 保证原子性（回滚）
- **锁/MVCC** 保证隔离性

## 二、并发问题与隔离级别

### 题目 3

事务 T1 读取了某行数据，事务 T2 随后修改了该行数据并提交，T1 再次读取同一行时发现结果已发生变化。这种现象称为（ ）。

A. 脏读  
B. 不可重复读  
C. 幻读  
D. 丢失更新

### ✅ 答案：B

### 📖 解析

| 并发问题 | 描述 | 区别 |
|----------|------|------|
| 脏读 | 读到了别人**未提交**的数据 | 数据还没提交就被读取 |
| 不可重复读 | 同一事务两次读同一行，结果**不同** | 已提交的更新导致 |
| 幻读 | 同一事务两次查询，返回**行数不同** | 已提交的插入/删除导致 |
| 丢失更新 | 两个事务各自修改，后者覆盖前者的修改 | 写写冲突 |

- 题干：T2 已**提交**，T1 读了两次结果不同 → **不可重复读**（针对的是同一行的更新）

---

### 题目 4（陷阱题）

下列隔离级别中，能够防止幻读的是（ ）。

A. 读未提交  
B. 读已提交  
C. 可重复读  
D. 串行化

### ✅ 答案：D

### 📖 解析

| 隔离级别 | 防止脏读 | 防止不可重复读 | 防止幻读 |
|----------|----------|---------------|---------|
| 读未提交 | ✗ | ✗ | ✗ |
| 读已提交 | ✅ | ✗ | ✗ |
| 可重复读 | ✅ | ✅ | ✗（标准SQL下） |
| 串行化 | ✅ | ✅ | ✅ |

- **注意**：MySQL InnoDB 的可重复读通过 MVCC 在一定程度上解决了幻读，但标准 SQL 定义中可重复读不防幻读，考试按标准 SQL 定义答题

## 三、数据库范式

### 题目 5（范式判断）

关系模式 R(学号, 课程号, 成绩, 教师姓名, 教师职称)，主键为（学号, 课程号）。若"教师职称"依赖于"教师姓名"而非主键，则该关系模式违反了（ ）。

A. 第一范式（1NF）  
B. 第二范式（2NF）  
C. 第三范式（3NF）  
D. BC 范式（BCNF）

### ✅ 答案：C

### 📖 解析

- **传递依赖**：教师职称 → 教师姓名 → (学号, 课程号)，这就是通过非主属性间接依赖主键
- **3NF** 要求消除传递依赖：非主属性不能通过另一个非主属性依赖主键
- 本题中存在"教师职称"→"教师姓名"的传递依赖，违反 3NF

### 🎯 范式口诀

```
1NF：列不可分（原子性）
2NF：消除部分依赖（非主属性完全依赖主键）
3NF：消除传递依赖（非主属性直接依赖主键）
BCNF：每个决定因素都是候选键
```

### 题目 6

关系模式 R(学号, 课程号, 成绩, 学生姓名)，主键为（学号, 课程号）。若"学生姓名"只依赖于"学号"，不依赖于整个主键，则该关系模式满足（ ），但不满足（ ）。

A. 1NF；2NF  
B. 2NF；3NF  
C. 3NF；BCNF  
D. 1NF；3NF

### ✅ 答案：A

### 📖 解析

- 学生姓名只依赖学号（主键的一部分），这是**部分依赖**
- 部分依赖违反 **2NF**（2NF 要求消除部分依赖）
- 每列数据原子性没有问题，满足 **1NF**
- 因此满足 1NF，但不满足 2NF

## 四、关系运算与 SQL

### 题目 7

下列 SQL 操作中，属于**关系运算中"选择"操作**的是（ ）。

A. `SELECT name FROM students`  
B. `SELECT * FROM students WHERE age > 18`  
C. `SELECT s.name, c.title FROM students s JOIN courses c ON s.id = c.student_id`  
D. `SELECT DISTINCT department FROM employees`

### ✅ 答案：B

### 📖 解析

| 关系运算 | 操作方向 | SQL 对应 |
|----------|----------|----------|
| 选择（Selection） | 行方向：筛选满足条件的行 | WHERE 子句 |
| 投影（Projection） | 列方向：选择指定的列 | SELECT 字段列表 |
| 连接（Join） | 横向合并两张表 | JOIN |

- B 中 `WHERE age > 18` 是按条件筛选行 → **选择**
- A 只选了 name 列 → **投影**
- C 连接两张表 → **连接**

### 题目 8（索引考点）

以下关于数据库索引的描述，**错误**的是（ ）。

A. 索引可以提高查询速度  
B. 索引会占用额外的存储空间  
C. 索引越多查询效率越高  
D. 索引会降低插入、更新、删除操作的性能

### ✅ 答案：C

### 📖 解析

- C 错误：索引并非越多越好，索引过多会：①占用大量存储；②降低写操作性能；③优化器选择成本增加
- A、B、D 均正确：索引提高查询速度、占用存储、降低写性能

## 考点总结

### ACID 速查表

| 特性 | 关键词 | 保障手段 |
|------|--------|---------|
| 原子性 | 全做或全不做 | Undo Log、回滚 |
| 一致性 | 数据正确、约束不破坏 | 完整性约束、正确逻辑 |
| 隔离性 | 并发互不干扰 | 锁、MVCC |
| 持久性 | 提交后永久 | Redo Log、落盘 |

### 并发问题识别

| 现象 | 原因 | 识别关键词 |
|------|------|-----------|
| 脏读 | 读到未提交数据 | "未提交就读到了" |
| 不可重复读 | 两次读同一行结果不同 | "已提交的更新，同行两次读不同" |
| 幻读 | 两次查询行数不同 | "已提交的插入/删除，行数变了" |
