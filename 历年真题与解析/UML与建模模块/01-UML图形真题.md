# UML 建模真题精选与解析

> UML 图形题每年出题 2–4 题，核心考法：给出描述判断用哪种图，或识别图中关系的含义。
> 难点在于：顺序图 vs 通信图、用例图关系、类图关系的辨析。

---

## 一、UML 图的选用

### 题目 1（经典高频）

需要描述系统中多个对象之间**按时间顺序**的消息交互过程，应选用（ ）。

A. 类图  
B. 用例图  
C. 顺序图  
D. 状态图

### ✅ 答案：C

### 📖 解析

| UML 图 | 适用场景 | 关键词 |
|--------|---------|--------|
| 用例图 | 系统功能边界、参与者与用例关系 | 参与者、功能、边界 |
| 类图 | 类的属性、方法及类间关系 | 类、属性、继承、关联 |
| 顺序图 | 对象间消息的**时间顺序** | 时间轴、消息顺序 |
| 通信图 | 对象间消息的**连接关系** | 对象链接、协作关系 |
| 状态图 | 对象在**生命周期内的状态变化** | 状态转换、触发条件 |
| 活动图 | 业务流程、并发控制流 | 活动、分支、并发 |
| 部署图 | 软件组件到物理节点的映射 | 服务器、硬件、节点 |
| 组件图 | 系统的组件组织和依赖 | 组件、接口、依赖 |

### 题目 2

某系统需要描述"订单"对象在其生命周期内从"待付款"到"已付款"到"已发货"到"已完成"的状态转变过程，应选用（ ）。

A. 顺序图  
B. 活动图  
C. 状态图  
D. 类图

### ✅ 答案：C

### 📖 解析

- 题干关键词：**"状态转变""生命周期"**
- 状态图专门描述对象从一个状态到另一个状态的转变及触发条件
- **顺序图**：强调时间顺序的消息交互
- **活动图**：强调流程和控制流（类似流程图）

---

### 题目 3

需要描述系统的参与者（用户）和系统提供的各项功能之间的关系，应使用（ ）。

A. 类图  
B. 用例图  
C. 顺序图  
D. 部署图

### ✅ 答案：B

### 📖 解析

- 用例图的三要素：**参与者（Actor）+ 用例（Use Case）+ 系统边界**
- 用例图回答"谁能用这个系统做什么"，是需求分析阶段的核心图

### 题目 4

以下描述与 UML 图类型对应，**错误**的是（ ）。

A. 活动图：适合描述并行处理和业务工作流程  
B. 组件图：描述软件组件及其依赖关系  
C. 部署图：适合描述系统对象在生命周期内的状态变化  
D. 顺序图：强调消息发送的时间顺序

### ✅ 答案：C

### 📖 解析

- C 错误：描述"状态变化"应该用**状态图**，不是部署图
- 部署图描述的是软件组件到物理硬件的映射关系

## 二、类图中的关系辨析

### 题目 5（关系识别）

在 UML 类图中，"汽车"类和"轮子"类之间，"汽车"由四个"轮子"组成，且轮子不能脱离汽车单独存在。这种关系是（ ）。

A. 关联关系  
B. 聚合关系  
C. 组合关系  
D. 依赖关系

### ✅ 答案：C

### 📖 解析

| 关系类型 | 含义 | 关键区别 | 图示符号 |
|----------|------|---------|---------|
| 依赖 | 一个类用到另一个类（临时） | 最弱，临时使用 | 虚线箭头 |
| 关联 | 一个类持有另一个类的引用 | 长期持有 | 实线 |
| 聚合 | 整体-部分，部分可独立存在 | 弱拥有，空心菱形 | 空心菱形 |
| 组合 | 整体-部分，部分**不能**独立存在 | 强拥有，实心菱形 | 实心菱形 |
| 继承（泛化） | 子类继承父类 | is-a 关系 | 空心三角 |
| 实现 | 类实现接口 | implements | 虚线空心三角 |

- "轮子不能脱离汽车单独存在" → **组合关系**（强整体-部分）
- 若轮子可以独立存在（如轮毂拿下来还是个零件）→ 聚合关系

### 题目 6

在 UML 类图中，"学生"继承"人"，表示 is-a 关系。这种关系的图示是（ ）。

A. 实心菱形 + 实线  
B. 空心菱形 + 实线  
C. 空心三角形 + 实线（箭头指向父类）  
D. 虚线 + 空心三角形（箭头指向接口）

### ✅ 答案：C

### 📖 解析

- 继承/泛化：空心三角 + 实线，箭头从子类指向父类
- 实现接口：虚线 + 空心三角，箭头从类指向接口
- 组合：实心菱形在整体端
- 聚合：空心菱形在整体端

## 三、用例图关系辨析

### 题目 7

在用例图中，"提交订单"用例中**必须**执行"验证用户身份"，这种关系是（ ）。

A. include（包含）关系  
B. extend（扩展）关系  
C. 泛化关系  
D. 关联关系

### ✅ 答案：A

### 📖 解析

| 用例关系 | 含义 | 关键区别 |
|----------|------|---------|
| include（包含） | 基用例**必须**执行被包含的用例 | 必须执行，无条件 |
| extend（扩展） | 在某种**条件下**，扩展用例才被执行 | 可选执行，有条件 |
| 泛化 | 子用例继承父用例的行为 | is-a 关系，可替换 |

- "必须执行" → **include**（包含）
- "在某条件下才执行" → **extend**（扩展）

### 实例辨析

```
include 示例：
"提交订单" --include--> "验证身份"（每次都要验证，必须）

extend 示例：
"查看订单" --extend--> "发送邮件通知"（可选功能，不是必须）
```

### 题目 8

在 UML 的 4+1 视图中，以下对应关系**错误**的是（ ）。

A. 逻辑视图 — 类图  
B. 开发视图 — 组件图  
C. 进程视图 — 部署图  
D. 物理视图 — 部署图

### ✅ 答案：C

### 📖 解析

| 视图 | 常用 UML 图 |
|------|------------|
| 逻辑视图 | 类图、状态图 |
| 开发视图 | 组件图、包图 |
| 进程视图 | 顺序图、活动图 |
| 物理视图 | **部署图** |
| 场景视图 | 用例图 |

- C 错：进程视图用的是**顺序图/活动图**，部署图是物理视图
- D 正确：物理视图 = 部署图

## 考点总结

### UML 图选用速查

| 题干关键词 | 选用的 UML 图 |
|-----------|-------------|
| 参与者、系统功能边界 | 用例图 |
| 类、属性、关系、继承 | 类图 |
| 时间顺序、消息交互 | 顺序图 |
| 对象协作、连接关系 | 通信图 |
| 状态变化、生命周期 | 状态图 |
| 业务流程、并发、分支 | 活动图 |
| 硬件节点、软件部署 | 部署图 |
| 组件、模块、依赖 | 组件图 |

### 类图关系识别

| 场景描述 | 关系类型 |
|---------|---------|
| "A 是 B 的一种"（is-a） | 继承/泛化 |
| "A 实现了接口 B" | 实现 |
| "A 拥有 B，B 可以独立存在" | 聚合 |
| "A 拥有 B，B 不能独立存在" | 组合 |
| "A 短暂使用了 B" | 依赖 |
| "A 长期引用 B" | 关联 |
