> 适用科目：软考系统架构设计师 · 案例分析（科目二）

## 二、答题通用原则

### 1. 格式规范（阅卷老师友好）

- **分点作答**：用①②③或(1)(2)(3)编号
- **先结论后解释**：第一句给出核心答案，后面补充理由
- **关键词加粗/下划线**：让阅卷者快速定位得分点
- **表格对比**：对比类题目用简表更清晰
- **字数控制**：每个小问 100–200 字为宜，不要写散文

### 2. 答题万能思路

```
第一步：审题 → 找出题目要求（让你做什么？选择/分析/评价/设计？）
第二步：定位知识点 → 确定考的是哪个知识模块
第三步：套模板 → 用标准答题结构组织答案
第四步：扣题干 → 引用题目中的具体信息作为论据
第五步：检查 → 是否答全了所有小问，格式是否清晰
```

### 3. 得分技巧

- **多答不扣分**：不确定时多写一两个点
- **用专业术语**：尽量用教材原话而非口语
- **结合场景**：不要只背概念，要把概念和题目场景结合
- **注意分值分配**：5 分的题至少答 5 个得分点

## 四、使用建议

1. **先熟记模板结构**：考前把每种题型的答题框架背熟
2. **做真题套模板**：用历年真题练习套用模板
3. **总结个人模板**：在通用模板基础上形成自己的简化版
4. **考场先列提纲**：拿到题先在草稿纸列出关键点再写

> ⚠️ 模板是框架，不是万能答案。考试时务必结合题目具体场景作答！
