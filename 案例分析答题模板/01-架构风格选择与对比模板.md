# 架构风格选择与对比 — 案例答题模板

> 高频考法：给出某系统的业务场景和需求描述，要求选择合适的架构风格并说明理由，或对比两种架构风格的优缺点。

---

## 模板一：选择架构风格并说明理由

### 题目类型

> "根据以上需求，请为该系统选择合适的架构风格，并说明选择理由。"

### 答题框架

```
【答】
（1）推荐采用 _____ 架构风格。

（2）选择理由如下：
  ① 从需求匹配角度：该系统具有 [题干中的特征A]，而 XX 架构的核心特征正是 [对应特征]，能有效满足此需求。
  ② 从质量属性角度：该系统要求 [高性能/高可用/可扩展等]，XX 架构通过 [具体机制] 可以满足该质量属性。
  ③ 从技术实现角度：XX 架构支持 [某种技术手段]，与题目中提到的 [技术要求] 一致。
  ④ 从团队/成本角度（如题目提及）：XX 架构 [降低耦合/便于团队分工/降低维护成本]。

（3）不选择其他架构的原因（加分项）：
  - 不选 YY 架构：因为 [具体原因，如不满足某需求/带来某缺点]
  - 不选 ZZ 架构：因为 [具体原因]
```

### 实战示例

**题目场景**：某大型电商平台需要支持高并发访问、快速迭代上线新功能、各业务模块独立部署和扩展。

```
【答】
（1）推荐采用微服务架构风格。

（2）选择理由如下：
  ① 从需求匹配角度：该电商平台需要"各业务模块独立部署和扩展"，而微服务架构的核心特征正是服务自治、独立部署，能有效满足此需求。
  ② 从质量属性角度：系统要求高并发访问（性能需求），微服务架构通过服务级别的水平扩展和负载均衡可以满足高并发要求。
  ③ 从技术实现角度：微服务架构支持各服务独立技术栈和独立发布，满足"快速迭代上线新功能"的要求。
  ④ 从团队角度：微服务架构便于大团队按业务域分组独立开发，提升交付效率。

（3）不选择单体架构：因为单体架构难以支持独立部署和按模块扩展，且大规模团队协作时代码冲突频繁。
    不选择SOA架构：因为SOA依赖重量级ESB，不符合"快速迭代"的敏捷需求。
```

## 模板二：对比两种架构风格

### 题目类型

> "请对比 A 架构和 B 架构的优缺点，并说明该系统更适合哪种。"

### 答题框架

```
【答】
（1）A架构与B架构的对比：

| 比较维度 | A架构 | B架构 |
|----------|-------|-------|
| 核心特征 | ... | ... |
| 优点 | ①... ②... | ①... ②... |
| 缺点 | ①... ②... | ①... ②... |
| 适用场景 | ... | ... |

（2）该系统更适合采用 ___ 架构，原因如下：
  ① 题目场景中的 [需求X] 与 ___ 架构的 [特征Y] 匹配
  ② ___ 架构能更好地满足 [质量属性Z] 的要求
  ③ 相比另一种架构，___ 架构在该场景下避免了 [具体缺点]
```

### 常考对比组合速查

| 对比组合 | 核心差异 |
|----------|----------|
| B/S vs C/S | 部署维护 vs 交互体验 |
| 单体 vs 微服务 | 简单统一 vs 独立扩展 |
| SOA vs 微服务 | ESB集中 vs 去中心化 |
| 分层 vs 管道-过滤器 | 职责划分 vs 数据流转 |
| 集中式 vs 分布式 | 一致性强 vs 可扩展 |
| 事件驱动 vs 请求响应 | 异步解耦 vs 同步直接 |

## 模板三：分析架构风格的适用性

### 题目类型

> "该系统采用了XX架构，请分析该架构在本场景下的优势和可能存在的问题。"

### 答题框架

```
【答】
（1）XX架构在本系统中的优势：
  ① [结合题干特征] → [对应到架构优势]
  ② [结合题干特征] → [对应到架构优势]
  ③ [结合题干特征] → [对应到架构优势]

（2）XX架构在本系统中可能存在的问题：
  ① [问题描述]：由于XX架构的 [固有特征]，在本系统 [具体场景] 下可能导致 [具体问题]
  ② [问题描述]：...

（3）建议的改进措施（如题目要求）：
  ① 针对问题①，可采用 [解决方案] 来缓解
  ② 针对问题②，可通过 [解决方案] 来应对
```

## 高频考点备忘

### 各架构风格的标准优缺点（答题素材）

| 架构风格 | 优点 | 缺点 |
|----------|------|------|
| 分层架构 | ①职责清晰 ②易于维护和测试 ③支持并行开发 | ①性能开销（层间调用） ②过度分层增加复杂度 |
| 微服务 | ①独立部署 ②技术异构 ③故障隔离 ④弹性伸缩 | ①分布式复杂性 ②数据一致性挑战 ③运维成本高 |
| 管道-过滤器 | ①高内聚低耦合 ②支持复用 ③便于扩展处理环节 | ①不适合交互式系统 ②解析/序列化开销 |
| 事件驱动 | ①松耦合 ②异步高并发 ③可扩展 | ①调试困难 ②事件顺序难保证 ③最终一致性 |
| 黑板架构 | ①适合非确定性问题 ②知识源可独立修改 | ①控制逻辑复杂 ②调试困难 ③性能难预测 |
| B/S架构 | ①零安装 ②统一维护 ③跨平台 | ①交互能力弱 ②对浏览器依赖大 ③安全性挑战 |
| C/S架构 | ①交互体验好 ②可利用客户端资源 | ①部署维护成本高 ②版本管理困难 |

## 答题注意事项

1. **必须结合题干**：不能只背课本内容，一定要引用题目中的具体描述
2. **理由要充分**：一般每个理由 1-2 句话，包含"因为...所以..."的逻辑
3. **注意分值**：如果是 10 分的题，至少准备 5 个得分点
4. **对比题用表格**：节省时间且清晰
5. **不确定时多写**：把能想到的相关点都写上，踩点给分
